## 队列管理

```
┌─────────────────────────────────────┐
│          JobQueue                   │
├─────────────────────────────────────┤
│  queue: deque()                     │
│    │                                │
│    ├─ JobInfo 1 (pending)           │
│    ├─ JobInfo 2 (pending)           │
│    ├─ JobInfo 3 (pending)           │
│    └─ ...                           │
│                                     │
│  submitted_jobs: dict               │
│    │                                │
│    ├─ "12345": JobInfo (submitted)  │
│    ├─ "12346": JobInfo (submitted)  │
│    └─ ...                           │
│                                     │
│  failed_jobs: list                  │
│    │                                │
│    ├─ (JobInfo, reason)             │
│    ├─ (JobInfo, reason)             │
│    └─ ...                           │
└─────────────────────────────────────┘

操作:
  add_job()          → 添加到 queue
  get_next_job()     → 从 queue 弹出
  mark_submitted()   → 移到 submitted_jobs
  mark_failed()      → 移到 failed_jobs
```

## 总结

该流程图展示了自动提交系统的：

1. **整体架构**: 各模块关系和数据流
2. **详细流程**: 从初始化到完成的完整过程
3. **资源检查**: 如何监控和限制资源使用
4. **脚本解析**: 如何提取作业信息
5. **作业提交**: 如何执行sbatch命令
6. **队列管理**: 如何管理不同状态的作业
7. **配置影响**: 各参数如何影响系统行为
8. **时间线**: 实际运行的时间序列
9. **状态转换**: 作业的生命周期
10. **错误处理**: 如何处理失败情况

通过这些图表，可以清晰理解系统的工作原理和设计思路。
